Hyperion API Examples

The Hyperion API enables developers to manage infrastructure programmatically. Here are 10 examples of how it accomplishes various tasks:

API Examples

Example 1: Provisioning a virtual machine instance:

POST /hyperion/api/v1/instances
{
  "name": "example-instance",
  "type": "t2.micro",
  "image": "ami-0c55b159cbfafe1f0"
}

Example 2: Creating a virtual private cloud (VPC):

POST /hyperion/api/v1/networks
{
  "name": "example-vpc",
  "cidr": "10.0.0.0/16"
}

Example 3: Configuring a security group:

POST /hyperion/api/v1/security-groups
{
  "name": "example-security-group",
  "rules": [
    {
      "type": "inbound",
      "protocol": "tcp",
      "port": 80,
      "source": "0.0.0.0/0"
    }
  ]
}

Example 4: Setting up an object storage bucket:

POST /hyperion/api/v1/object-storage
{
  "name": "example-bucket",
  "acl": "private"
}

Example 5: Defining an identity and access management (IAM) role:

POST /hyperion/api/v1/iam-roles
{
  "name": "example-role",
  "permissions": [
    "read",
    "write"
  ]
}

Example 6: Launching a database instance:

POST /hyperion/api/v1/databases
{
  "name": "example-db",
  "type": "mysql",
  "version": "5.7",
  "storage": "50GB"
}

Example 7: Configuring a load balancer:

POST /hyperion/api/v1/load-balancers
{
  "name": "example-lb",
  "type": "application",
  "listeners": [
    {
      "port": 80,
      "protocol": "HTTP"
    }
  ],
  "targets": [
    {
      "name": "target-group-1",
      "port": 80
    }
  ]
}

Example 8: Defining a DNS record:

POST /hyperion/api/v1/dns
{
  "name": "example.com",
  "type": "A",
  "value": "1.2.3.4"
}

Example 9: Configuring auto-scaling:

POST /hyperion/api/v1/auto-scaling
{
  "name": "example-asg",
  "min_instances": 2,
  "max_instances": 5,
  "target_cpu_utilization": 70
}

Example 10: Setting up monitoring and logging:

POST /hyperion/api/v1/monitoring
{
  "name": "example-monitoring",
  "type": "cloudwatch",
  "metrics": ["CPUUtilization", "NetworkIn", "NetworkOut"],
  "logs": ["app-logs", "system-logs"]
}
